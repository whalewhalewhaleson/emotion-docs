## How to read a project

| Section         | What to look for                                          |
| --------------- | --------------------------------------------------------- |
| Problem         | The real workflow pain before the tool existed.           |
| Build decisions | The product, design, and technical choices that mattered. |
| Results         | What changed after the system went live.                  |
| Lessons         | The reusable pattern that carries into the next build.    |
